# Security Policy

## Supported versions

Only the latest release receives security fixes.

| Version | Supported |
|---------|-----------|
| 0.1.x   | Yes       |

## Reporting a vulnerability

**Please do not file a public GitHub issue for security vulnerabilities.**

Report them privately by emailing the maintainer or by using
[GitHub's private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing/privately-reporting-a-security-vulnerability)
if enabled on this repository.

Include as much of the following as you can:

- A description of the vulnerability and its potential impact
- Steps to reproduce or a proof-of-concept
- Affected versions
- Any suggested mitigations you are aware of

## What to expect

- **Acknowledgement** within 3 business days
- **Status update** (confirmed, not reproducible, or out of scope) within 10 business days
- A patched release and public disclosure once a fix is ready

## Scope

This project is a starter template. In a production deployment, additional
hardening is expected (authentication, rate limiting, HTTPS termination, secret
management). Issues in those areas that are inherent to the template design are
in scope; issues in your own deployment configuration are not.
